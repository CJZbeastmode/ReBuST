Because **STOP is an absorbing, low-variance, immediately rewarding action**, and **DQN is a greedy value learner**.
That combination is lethal in your environment.

Here’s the precise mechanism—no fluff.

---

## 1. STOP is an *absorbing action* with clean targets

In your env:

* `STOP` → `done=True`
* Target:

  ```
  Q(stop) ≈ reward
  ```
* No bootstrapping
* No uncertainty
* No delayed credit

For `ZOOM`:

```
Q(zoom) ≈ reward − λ + γ · max_a Q(s')
```

That second term is:

* noisy
* poorly estimated early
* often near zero because episodes are short

So **STOP always has a lower-variance, faster-converging TD target**.

DQN *loves* that.

---

## 2. Early overestimation of STOP is self-locking

Once this happens even once:

```
Q(stop) > Q(zoom)
```

you are basically done.

Why?

```python
action = argmax(Q)
```

So:

* STOP is selected more
* ZOOM transitions disappear from replay
* Q(stop) variance ↓
* Q(zoom) stays noisy / undertrained

This is a **positive feedback loop**.

A2C never enters this loop because it samples from a distribution.

---

## 3. Lambda makes STOP *dominant*, not balanced

You intend lambda as a *soft budget constraint*.

But in DQN it becomes a **hard action filter**.

Example:

| Action     | Raw reward | Cost | λ   | Effective |
| ---------- | ---------- | ---- | --- | --------- |
| STOP       | 1.0        | 0    | 1.0 | **1.0**   |
| ZOOM       | 2.0        | 1    | 1.0 | **1.0**   |
| ZOOM (bad) | 1.0        | 1    | 1.0 | **0.0**   |

So DQN learns:

> “STOP is never worse, often better, and safer.”

A2C *still samples zoom* because probability ≠ value.

---

## 4. Short horizon kills delayed value

Your setup:

```python
max_steps = 10
gamma = 0.99
```

In practice:

* Zoom benefit often appears at step 2–4
* But episode frequently terminates early anyway

So DQN sees:

```
reward_now(zoom) < reward_now(stop)
```

and does not care about hypothetical future states.

This is textbook **myopic value collapse**.

---

## 5. Replay buffer amplifies imbalance

Replay buffer ≠ neutral memory.

It is **policy-conditioned**.

Once STOP dominates:

* 80–90% of buffer = STOP transitions
* Zoom Q never stabilizes
* Target network reinforces STOP dominance

This is why:

> epsilon ≠ exploration once learning starts

---

## 6. Why A2C does *not* collapse

A2C updates:

```
∇ log π(a|s) · (R − V(s))
```

Even if:

```
V(stop) > V(zoom)
```

zoom still has **non-zero probability mass**.

That means:

* zoom keeps getting sampled
* advantages stay meaningful
* lambda does not hard-block zoom

This is the key difference.

---

## 7. This is a *known* DQN failure mode

What you’re seeing is a combination of:

* Absorbing actions
* Early termination
* Action costs
* Short episodes

This exact failure appears in:

* budgeted MDPs
* option-termination learning
* anytime decision processes

DQN is the **wrong tool** unless modified.

**DQN collapses to STOP because STOP is easier to learn, safer, terminal, and immediately rewarding.**

A2C survives because:
